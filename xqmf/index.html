xqmf: Extensible quality metrics framework

[ what's new ] [ project ] [ discuss ] [ download ]
Software maintainability (or lack thereof) has paradoxically been one of the criticisms levelled against open-source projects. As no-one is responsible for patrolling code quality, it can get rather opaque. So maintainability is impaired (never mind that closed-source isn't maintainable by definition, we don't want logic here, it's a holy war.)

At any rate, that's gonna change. The metrics framework you see a-birthing will make it easy to set up an ongoing code quality monitoring process, which is integrated with CVS if you use it, and which is integrated with an optional database, so that reporting will be easy. You saw it here first.
STATUS / NEWS
9 June 2000 Created this page and thereby officially started development.
30 July 2000 OK, so now maybe things are really underway. I have the first milestone complete: a framework implementation which writes to XML files and to an optional database. Of course, the initial metric is simply lines of code... Anyway, here's where you can download what I've got so far. Give it a shot and tell me what you think.
