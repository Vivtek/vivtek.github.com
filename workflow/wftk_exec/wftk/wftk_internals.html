Here are a few things that the library functions will use (and adaptor code as well) but that don't really need to be exposed to the cold, hard world. First are the IDs for the types of adaptor.
  
#define DSREP         0
#define PDREP         1
#define USER          2
#define DB            3
#define NOTIFY        4
#define SCRIPT        5
#define DATATYPE      6
#define DEBUG         7
#define CONFIG        8
And the other is the definition of the WFTK_ADAPTOR structure used to pass APIs back and forth. Sort of a poor man's ActiveX, I guess.
  
typedef struct _wftk_adaptor WFTK_ADAPTOR;
struct _wftk_adaptor {
   int dummy;
};
