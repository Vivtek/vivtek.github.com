Where did my Google go?
Provision of libraries (2009)

In the context of looking at different languages for development of the wftk, I eventually ended up with Perl because I like the CPAN so very much. It simplifies everything, because instead of getting stuck wishing I had a shell for quick testing and poking around, I just perl -MCPAN -e shell and install Term::Shell and ... I have a shell suitable for dropping into my package. And it's like that for everything!

But then I hear about all this excitement over Ruby on Rails. Hmm, I think. Wouldn't it be nice to see what all that's about? But ... I wouldn't have CPAN. There are Ruby Gems, though. Sorta the same thing. Right?

So I thought it might be a good idea to compare the world's current major languages in terms of library/module repositories, both as a producer and a consumer of modules. This matrix is the result of that research.
