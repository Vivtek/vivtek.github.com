This page is a central point for the wftk code documentation. Some explanation is in order. The wftk is coded in a literate style, which means that the source code and its documentation are maintained together. The source files are XML documentation which are processed by means of my little tool here.

The code for the wftk library is here. The way it works is that the core library defines the workflow API and implements the basic logic, while all interaction with other systems is put into what I call adaptors. An adaptor encapsulates everything the wftk needs to know about a system (for instance, a datasheet storage repository, where process information is kept). Then you can implement an adaptor which talks to any arbitrary system, and the wftk should work transparently with it.

Adaptors fall into classes, and each class has its own subdirectory in the main wftk code directory, so you can browse around to see what I mean.

Included with the wftk_lib library is also a rudimentary command-line interface to allow you to do things from the shell. It doesn't expose the full wftk API, though. Too much work to think it through.
