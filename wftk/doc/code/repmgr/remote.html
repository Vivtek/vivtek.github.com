#include "repmgr.h"
#include "time.h"
#ifdef WINDOWS
#include "windows.h"
#include "winsock.h"
#else
#include "sys/socket.h"
#include "netinet/in.h"
#include "netdb.h"
#include "fcntl.h"
#endif
#include "stdarg.h"
Here's our namespace magic macro.
  
#ifndef REPMGR_NAMESPACE
#define REPMGR_NAMESPACE
#endif

#define FN_NAME(x) CONCAT(REPMGR_NAMESPACE,x)
#define CONCAT(x,y) CONCAT2(x,y)
#define CONCAT2(x,y) x ## y
This uses C preprocessor concatenation to tack the value of "REPMGR_NAMESPACE" onto the beginning of each function name. This is defined on the command line used to compile the file. Getting this to work right took a couple of tries, because the C preprocessor didn't want to resolve REPMGR_NAMESPACE in the concatenation. The naive approach (#define FN_NAME(x) REPMGR_NAMESPACE ## x) just kept defining REPMGR_NAME_SPACErepos_list and stuff. You can see how I finally persuaded the preprocessor to do what I wanted.
