Up to now, there has been no remote publishing because the main library doesn't call up remote repositories for publishing. This may make it easier to manage this, but only after the stuff is actually, you know, implemented. WFTK_EXPORT int FN_NAME(repos_publish_all) (XML * repository) { return 0; } WFTK_EXPORT int FN_NAME(repos_publish_list) (XML * repository, const char * list) { return 0; } WFTK_EXPORT int FN_NAME(repos_publish_obj) (XML * repository, const char * list, const char * key) { return 0; } I probably cut out 20% of the weight of the repmgr library just by eliminating the publishing logic.
