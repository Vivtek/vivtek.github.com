wftk_session: wftk context handling

[ download ] [ xml source ] [ discussion ]

The context handling machinery of the wftk consists of the session API, the configuration API, and the adaptor mechanism. This has all been sort of disorganized up until now (March 2002) and I'm sure that it will continue to be disorganized henceforth, but hopefully a teensy bit less so.

  • Include file (and API definition)
  • Function definitions
  • Working with sessions
  • wftk configuration module
  • wftk adaptor code
    • wftk_get_adaptor(): building an adaptor
    • wftk_free_adaptor(): disposing of an adaptor
    • wftk_call_adaptor(): calling an adaptor function
    • wftk_get_adaptorlist(): getting a list of default listeners
    • wftk_free_adaptorlist(): freeing an adaptor list
    • wftk_call_adaptorlist(): shotgun notification
