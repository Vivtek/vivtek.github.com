wftk: wftk command-line utility

[ download ] [ xml source ] [ discussion ]
The command-line utility is first of all a convenient way to test the library after compilation. But in addtion to that, it provides a dandy way for shell scripts to interact with workflow (not the least of which is to enable scripts to start processes. Cool stuff, man.) It also lets you interface wftk with anything that can do a system call.

Here's how we do all this stuff:
  • wftk command-line utility
  • main()
    • The list of commands in printf format
  • Dealing with processes
    • create: Creating a new datasheet
    • delete: Deleting an existing process
    • define: Attach a procdef
    • start: Start a procdef
    • show: Displaying process information
    • adhoc: Running ad-hoc workflow
  • Working with tasks
    • task: Show a task, including data requirements
    • tasks: List tasks
    • todo: List indexed tasks
    • complete: Complete a task (or start the process)
    • reject: Reject a task
    • newtask: Create an ad-hoc task
    • rescind: Delete an ad-hoc task
    • assign: Attach a user to a task
  • Working with requests
    • request: Show a request
    • requests: List requests
    • accept: Accept a request
    • decline: Decline a request
    • ask: Make a request
    • forget: Rescind a request
  • Dealing with values
    • set: setting named values
    • get: getting named values
    • values: listing named values
    • html: getting HTML for a named value
    • htmlblank: getting a blank field
  • Working with the enactment history
    • log: get or write to enactment history
  • Working with roles
    • roles: list roles in a datasheet
    • role: display or assign a role
  • Working with users
    • users: list users in a datasheet
    • user: display user or set attribute
  • action: Performing actions with possible deferment
