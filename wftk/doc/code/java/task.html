wftk-j: Modelling tasks

[ Previous: Modelling list entries ] [ Top:  ] [ Next: Users in the system ]
A task, as far as the client is concerned, is an entry with a little extra functionality. We can construct a blank Task object and then add it to the system as an ad-hoc task, assign it to a user, or assign it as an ad-hoc task to an existing process (Entry object).
  
package org.wftk;
public class Task extends Entry {
   public Task (Repository r, String list, String key_in) {
      rep = r;
      list_id = list;
      key = key_in;
   }

   public boolean is_task() { return true; }
   public void complete () {
   }
}
