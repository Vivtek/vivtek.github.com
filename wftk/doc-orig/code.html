wftk: Design and implementation

[ wftk documentation home ]
The wftk is implemented as a library in ANSI C. Under Windows, it is (optionally) a DLL, but the Unix makefiles still just produce a static-link library. Its internal data structures are based on XML and use a library called the XMLAPI. The XMLAPI library uses James Clark's expat parser to read XML files and data; there's no reason you couldn't write a simple wrapper for any other parser, though. (But why would you want to? Expat works!)

A wftk application is organized into the following levels:

  • The front end (e.g. thick client, Web server, etc.: the application)
  • The core engine
  • A series of adaptors through which wftk talks to other systems.

As of July 2001, there is only one front end of a sort: the command-line wrapper, which is a standalone C program which links to the wftk library. Work has begun on a Python extension which will allow the coding of front ends in Python (GUI work with wxPython, or Web work with Zope), and similar front ends are planned for Perl, PHP, and Tcl (for AOLserver).

The adaptors fall into classes; each adaptor class addresses one kind of interaction with the outside world. Examples are message sending, datasheet storage, and database interaction. Each class has at least one adaptor implemented, and several have more than one (for example, there are working database adaptors for ODBC and Oracle, and working datasheet storage adaptors for the local filesystem and for arbitrary databases.)

There is now (July 28, 2001) the beginnings of a Python wrapper; to date, this consists of a prety good wrapper for XMLAPI, the main library to follow. Full Python integration will require quite a bit of thought, as will integration with any scripting language.

The basic layout of the source code is as follows:

  • WFTK code (core engine and command-line wrapper)
    • Main library implementation (wftk_lib.xml)
    • Adaptor interface (wftk_adaptors.xml)
    • Configuration interface (wftk_config.xml)
    • Command-line interface (wftk_util.xml)
    • (subdirectories): One subdirectory for each adaptor class.
  • XMLAPI code
  • Python wrapper
