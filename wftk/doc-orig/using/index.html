Using the wftk

Purpose of this document

This User's Guide documents how to use the wftk end-user tools to get workflow done. The only problem with that is that as of this writing (May 14, 2001) there is only one end-user tool, and that's the command-line utility, which I really wrote just for testing.

However, it still makes sense to start blocking out the major things you can do with wftk, and as the CGI and other versions become available, they can be filled in. (And in the meantime, if you're doing workflow from the command line or with shell scripts, you can still see how things should work.)

If you're embedding the wftk library into your own application, you need to see the Programmer's Guide.

Basic categories of workflow activity

The purpose of workflow is of course to organize tasks so that you can complete them. In order to do that, of course, you have to create processes, create tasks, define process templates, and so forth. Each category of task has its own chapter:
  • Creating and manipulating processes
  • Creating and assigning tasks
  • Getting and setting values
  • Completing or rejecting tasks
  • Defining workflow templates (procdefs)
  • Working with ad-hoc workflow
  • Getting overviews and reports
