xmlapi: XML manipulation library

[ download ] [ xml source ] [ discussion ]
This is another spinoff of the open-source workflow toolkit. expat is great for parsing XML files -- but what do you do with them once parsed? For the wftk and for other projects, I needed a flexible set of functions which could load an XML data structure into memory and then do things to it. This library is the result.

(08/11/00) One of the things I did to it was to write the XMLCGI interface to CGI. It takes all the salient information about the current CGI environment and sticks it into an XMLAPI structure. Makes it easy to do CGI in C. At least for me.

  • Definitions
  • Functions except for xml_read
  • xml_write: Writing XML data to disk
  • xml_string: Writing XML data to strings in memory
  • xml_prepend: Inserting elements
  • xml_append: Inserting elements
  • xml_replace and xml_replacecontent: Replacing an element with another.
  • Bookmarking things: xml_loc and xml_getloc
  • Working with attributes: xml_set and xml_attrval
  • xml_create: Creating an empty element
  • xml_createtext: a shortcut for plain text
  • xml_free: Cleaning up afterwards
  • Deleting pieces: xml_delete
  • Children: xml_first and xml_last
  • Siblings: xml_next and xml_prev
  • inserting things: xml_insertbefore and xml_insertafter
  • xml_copy: making fresh copies of XML
  • xml_attr*: Working with attributes
  • xml_read: Using expat to parse XML files into memory
    • Handling elements: startElement
    • Handling elements: endElement
    • Handling non-element data: charData
